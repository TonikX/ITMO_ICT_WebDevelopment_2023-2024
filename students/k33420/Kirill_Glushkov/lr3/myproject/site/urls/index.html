Отчёт по URL конфигурации Django проекта

Следующий список предоставляет информацию о путях (URLs), доступных в одном из приложений Django проекта. Каждый путь связан с соответствующим видом (view), который обрабатывает HTTP-запросы по этому адресу.

Пути и Соответствующие Виды

  1. Стоимость по дате и доктору
  2. URL: cost-by-date/
  3. Вид: getCostByDateNDoctor

  4. Имя: cost-by-date

  5. Пациенты по дате

  6. URL: patients-by-date/
  7. Вид: getPatientsByDate

  8. Имя: patients-by-date

  9. Список оплативших пациентов

  10. URL: list-patients-paid/
  11. Вид: getListPatientsPaid

  12. Имя: list-patients-paid

  13. Доктора

  14. URL: doctors/
  15. Вид: getDoctors

  16. Имя: doctors

  17. Пациенты по имени доктора

  18. URL: doctor-patients/
  19. Вид: getPatientsbyDoctorName

  20. Имя: doctor-patients

  21. Количество пациентов оториноларинголога

  22. URL: numbers-of-patients-otorhinolaryngologist/
  23. Вид: getNumbersOfPatientsOtoronginolog

  24. Имя: numbers-of-patients-otorhinolaryngologist

  25. Доктор по дням

  26. URL: doctor-by-day/
  27. Вид: getDoctorByDay

  28. Имя: doctor-by-day

  29. Пациенты

  30. URL: patients/
  31. Вид: getPatients

  32. Имя: patients

  33. Отчёт о визите

  34. URL: visit-report/
  35. Вид: getVisitReport
  36. Имя: visit-report

В конце конфигурационного файла используется функция format_suffix_patterns, чтобы добавить поддержку форматированных суффиксов к каждому из URL путей.

urlpatterns = format_suffix_patterns(urlpatterns)

Это позволяет клиентам запросить определённый тип контента, например JSON или HTML, используя префиксы, такие как .json или .html в конце URL пути.
